IP
・コネクションレス型　TCPはコネクションするが、IPではその機能がない。
・階層型アドレス方式　ネットワーク部とホスト部の2階層で構成されている。
・ベストエフォート　帯域の保証をしないため、通信量が多くなると速度が低下したりパケットが届かなくなることがある。TCPなら再送する機能がある。

ギャランティ型　帯域保証がある通信形態。専用線による通信がコレにあたる。

Null0インタフェース　ルーティングループを防ぐための仮想I/F

DHCPサーバ　クライアントに割り当てるアドレスはpoolで範囲を設定できるが、直接個数を入力するわけではない。
(config)# ip dhcp pool {プール名}
(dhcp-config)# network {ネットワークアドレス} {サブネットマスクorプレフィクス}　割り当てるアドレスの範囲
(dhcp-config)#lease {日数}　貸出期間の設定
(dhcp-config)#dns-server [IPアドレス１] [IPアドレス２]　～～dnsサーバは８つまで指定可能。
(dhcp-config)#default-router [IPアドレス1] [IPアドレス２]　～～デフォルトゲートウェイは8つまで指定可能。
(dhcp-config)#ip dhcp excluded-address {開始アドレス} [終了アドレス]　割り当てから除外するアドレス範囲

再帰スタティックルート　転送先にネクストホップだけを指定する方法。
直接接続スタティックルート　転送先に出力インタフェースのみを指定する方法
※　なぜ再帰と呼ぶのか...ネクストホップが誤っている,存在しないアドレスだったとしても、自身のルーティングテーブルを参照して正しいネクストホップが確定する場合にはそれを利用するという挙動をするから。

VLSM (Variable Length Sunbnet Mask)　可変長サブネット

IPsec 複数のプロトコルから構成される。下記はあくまで一例。
・AH(Authentication Header)：データの認証を行う
・ESP(Encupsulated Security Payload)：データの認証と暗号化を行う
・IKE(Internet Key Exchange)：共有鍵の交換を安全に行う

show interfaceにて
runts　64バイトより小さいフレームを受信した場合に増加する。　duplexの不一致が原因と考えられる。
giants　1518バイトより大きいフレームを受信した場合に増加する。　NIC不良が増加の原因。
no buffer 受信したが、バッファが足りずに破棄したフレームの数。　ブロードキャストストームが原因。
ignored　バッファが足りずに無視したフレームの数。　ブロードキャストストームが原因。
input errors　runts,giants,no buffer,CRC,ignored,overrun,frameの合計数。

PuppetとChefはHTTPを利用する。AnsibleはSSH,NETCONFなどを利用する。

MPLS(Multi Protocol Label Switching) VPNやWANで使われる技術。L2とL3の間にヘッダを追加するため、L3ヘッダを参照せずにデータ転送できる。設置場所がMPLS網内ならPE(Provider Edge),拠点側ならCE(Consumer Edge)と呼ばれる。RE-CE間はアクセス回線と呼ばれ、使用回線に制限がない。
