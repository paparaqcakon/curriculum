ハードウェアクロック・・・ハードウェアが管理している時刻。RTC(Real Time Clock)とも呼ばれ、PC起動時ていなくても常に動作する。
システムクロック・・・Linuxカーネルが管理している時刻。PC起動時にハードウェアクロックを参照して時刻設定をする。
起動後は上記二つは別々に動作して同期を取らない。

○date・・・システムクロックを参照し表示する。
$ date　040114051980　　のように、月/日/時/分/西暦と入力するとシステム黒くを変更できる。

○hwclock・・・ハードウェアクロックを参照する。{VMwareでは参照できなかった..H/Wの設定がないのか？}
オプションとして[-w]でH/WクロックをSクロックの時刻と同期、[-s]でSクロックをH/Wクロックの時刻と同期。


NTP　時刻設定をNTPサーバと同期する。前提としてntpdateモジュールと、apt-get設定が正しいことが必要。
○apt-getupdate・・・apt-getのパッケージファイルを更新する。{VMwareではできない...}
○apt-get install ntpdate・・・これでntpdateモジュールの導入ができる。{なんで..できないの？}

○ntpdate IPアドレス(サーバ名)・・・NTPサーバと同期を行う。{}

NTPサーバ側でも同様にntpdateモジュール導入する。
○apt-get install ntp・・・サーバでのntpモジュール導入コマンド。導入すると、ntpdが起動する。ただし、初期状態のため一旦ntpdプロセスを終了。
NTPサーバの設定はntp.confファイルで行う。（普通ファイルの場所は/etc/ntp.conf）
初期設定では「restrict default ignore」とntpサーバとの時刻同期をしない設定になっている。
$restrict 192.168.0.0 mak 255.255.255.0 nomodify notrap　このように入力してLAN内からの応答を許可。
