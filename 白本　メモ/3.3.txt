・スタティックルーティング(静的)　手動でルーティングテーブルを作成する。ルータ間で情報交換しないため、負荷は少ない。経路に障害が発生してもお構いなしにそのルートにデータ転送する。

○設定方法
hostname(config)# ip route <宛先ネットワークアドレス> <サブネットマスク> <ネクストホップもしくは出力インターフェース>　(必要なら<アドミニストレーティブディスタンス値>)

hostname(config)# ip route 192.168.200.0 255.255.255.0 192.168.100.2

hostname#show ip route 
　　〜〜〜〜中略〜〜〜〜
  S 192.168.200.0/24 [1/0] via 192.168.100.2
　　〜〜〜〜中略〜〜〜〜
スタティックルートのコードは「S」と表記される。viaはネクストホップのこと。この[1/0]がアドミニストレーティブディスタンス値。

※間違えて登録したら no ip route 192.168〜〜 で削除

・デフォルトルートの設定
hostname(config)# ip route 0.0.0.0 0.0.0.0 <ネクストホップ>

hostname# show ip route
  〜〜〜〜中略〜〜〜〜
　Gateway of last resort is 172.16.0.2 to network 0.0.0.0
　〜〜〜〜中略〜〜〜〜
　S* [1/0] via 172.16.0.2
　〜〜〜〜中略〜〜〜〜
デフォルトルートの設定をしていないと、Gateway of last resort is not set　と表示される。
デフォルトルートのコードは「*」
